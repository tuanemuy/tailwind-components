# サブカテゴリーのREADMEの整備

## 背景

- `src/examples/` 以下にHTML形式のコンポーネントのサンプル集を配置した
- `src/examples/components.json` にサンプル集の目次を配置した
- READMEを整備して、実装を確認しなくてもどのコンポーネントを参考にすべきかがわかるようにしたい

## タスク

- 各コンポーネントの用途をコンポーネント名と実装から分析する
- サブカテゴリーの `README.md` に各コンポーネントの説明を記載する

## 条件

- 指定されたサブカテゴリーを作業対象とする
- それぞれ以下の項目をフラットなテキスト（後にJSONにするため）で記載する
    - 説明
    - 主なユースケース
- ファイルが大きい場合があるため、コンポーネントごとに作業を行う
- あらゆるアプリケーションの実装者がサンプルを探す時に、どれを参考にすべきか迷わないような説明にする
